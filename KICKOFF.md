# 개발 착수 지시 — 수행 계획서 자동 작성 에이전트

작업 전 프로젝트 폴더의 `PRD.md`와 `template-spec.md`를 먼저 읽고, 그 내용을 기준으로 진행해줘.
기준 템플릿 파일은 `01.AgentGo PoC(가치 증명) 수행 계획서_v1.2.docx` 이다.

## 만들 것
필수 값 8개만 입력받아, 회사 표준 워드 템플릿(.docx)의 **서식을 보존한 채**
값만 채워 수행 계획서를 자동 생성하는, 클로드 코드(CLI)에서 동작하는 에이전트.

## 이번 단계 범위 (MVP)
1. **필수 값(A 그룹 8개) 입력받기** — `template-spec.md` 0장 A 표 기준
   client_name / start_date / total_weeks / poc_purpose / verify_tasks /
   integration_targets / solution_config / client_pm
   - 누락 항목은 되묻는다.
   - client_pm 등 연락처·개인정보는 마스킹 형태로 받는다(실명 대신 직무·등급 권장).
2. **내용 자동 작성** — `template-spec.md` 1장 매핑에 따라 [가변] 항목만 생성,
   [기본]은 기본값, [고정]은 템플릿에서 상속.
   - start_date + total_weeks로 5장 일정 주차·날짜를 자동 산출한다.
   - 일정·스펙 등 확인이 필요한 수치는 `[확인 필요]`로 표시한다.
3. **표준 양식(.docx) 출력** — 기준 템플릿의 표·스타일·서식을 깨지 않고
   값만 주입해 새 .docx로 저장한다. (텍스트 단순 치환 금지 — 표 셀/플레이스홀더 단위 주입)

(수정·재생성, 선택 입력 B 되묻기, 빠진 값 체크, HWP/PDF 출력은 다음 단계 — 지금은 만들지 않는다.)

## 제약·전제
- 외부 유료 API/도구 도입은 제한적 → .docx 처리는 오픈소스 라이브러리(예: python-docx) 전제.
- 개인정보·고객사 정보를 외부에 원문 노출하지 않는 설계를 우선한다.
- 표준 양식은 이미 확보됨(기준 템플릿). 새로 만들 필요 없음.

## 핵심 기술 과제 (먼저 검증할 것)
기준 템플릿이 **빈 양식이 아니라 특정 고객 건으로 채워진 사례 문서**다.
→ "고정 골격 / 사업별 가변 값"을 구분해, 가변 값만 안전하게 교체할 방법이 필요하다.
   잘못 남은 이전 값(예: 이전 고객사명·날짜)이 새 문서에 남지 않도록 한다.

## 진행 방식 (한 번에 다 만들지 말 것)
1. 먼저 기준 .docx를 파싱해 **가변 항목의 위치(표·셀·플레이스홀더)를 식별**하고,
   `template-spec.md` 1장 매핑과 일치하는지 점검 결과를 보여줘. 내 확인을 받아라.
2. 확인 후 입력→작성→.docx 출력 흐름을 구현하라.
3. 샘플 입력 1건으로 생성한 .docx가 서식 깨짐 없이 나오는지 검증한 뒤 보고하라.

먼저 1번(가변 항목 위치 식별 + 매핑 점검)부터 보여줘.
