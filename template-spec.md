# 템플릿 매핑 사양서 — 수행 계획서 자동 작성 에이전트

> 기준 템플릿: `01.AgentGo PoC(가치 증명) 수행 계획서_v1.2.docx`
> 목적: 입력 값(A/B/C)이 문서의 어느 섹션·표·셀에 주입되는지 명세한다.
> 분류 규칙 — **[가변]** 사업별 입력, **[기본]** 선택 입력(기본값 자동), **[고정]** 템플릿 상속.

---

## 0. 입력 값 정의

### A. 필수 입력 (없으면 작성 불가)
| 키 | 설명 | 예시 | 형식 |
|---|---|---|---|
| `client_name` | 고객사명 | 예시고객사 | text |
| `start_date` | 착수일 | 2026-05-29 | date |
| `total_weeks` | 총 수행 주차 | 6 | int |
| `poc_purpose` | PoC 목적·배경 (고객 업무 목표) | 업무 효율·완결을 위한 Agentic AI 검증 | text(1~2줄) |
| `verify_tasks[]` | 검증 과제 목록 | RAG 정합성, 기간계 연계, M365 연계 | list |
| `integration_targets` | 연동 대상 | SSO 종류 / 기간계 유무 / M365 유무 | object |
| `solution_config[]` | 솔루션 구성 (제품·User·LLM·vCPU) | Enterprise(llama4), Coder(10U, GPTOSS 120B), Flow(1.5~2 vCPU) | list |
| `client_pm` | 고객 측 PM·승인권자 (연락처 **마스킹**) | `예시고객사 PM [담당자 직급 / 010-****-**** / name@example.com]` | text |

> **`client_pm` 형식**: 클로잇 측 비상연락망 형식을 그대로 따른다 →
> `{소속} {직책} [{이름} {직급} / {연락처} / {이메일}]`. 연락처·이메일은 마스킹해 저장.

### B. 선택 입력 (기본값 자동, 협의 시 변경)
| 키 | 기본값 |
|---|---|
| `kpi_targets` | 정확성 95%↑, 사용성 4.0↑(5점) |
| `infra_os` | 1안 Ubuntu 22.04 LTS / 2안 Rocky Linux 8·9 |
| `use_nas` | 사용 (NAS 1TB↑) |
| `meeting_cycle` | 격주 |
| `phase_plan` | 6단계 표준 WBS (아래 5장 참조) |
| `author` | 문서 작성자 (개정이력) — 기본: 실행 사용자, 미입력 시 `[작성자]` |
| `approver` | 문서 승인자 (개정이력) — 기본: `[승인자]` |

### C. 고정값 (입력 불필요 — 템플릿 상속)
- 인프라 표준 스펙(HW/OS 표 골격, 포트·도메인 리스트)
- 클로잇 측 비상 연락망 골격 (PM·파트너)
- 서명란 양식, KPI 평가 영역 골격(정확성·연동성·사용성)

## 2. 처리 규칙
0. **고객사명 슬롯 일괄 치환**: 기준 템플릿에는 같은 고객사가 여러 표기로 박혀 있다
   (예: 표지 `ABC` / 2장 `ABC SSO` / 5장 `ABC INC` — 표기는 달라도 모두 동일 고객사를 의미).
   이 3곳은 전부 `client_name` 슬롯이므로 입력값으로 **함께** 치환한다.
   한 가지 철자만 단순 치환하면 2·5장이 누락되므로, 알려진 변형(공백 유무 등)을
   모두 슬롯으로 인식하거나 위치 기반으로 주입할 것. (`client_name`은 필수 입력 A#1)
1. **서식 보존**: 텍스트 단순 치환이 아니라 표 셀·플레이스홀더 단위로 값 주입 (표·스타일 유지).
2. **자동 산출**: `start_date` + `total_weeks` → 각 단계 종료일 계산. 주차 라벨(`N주차(~M/D)`) 자동 생성.
3. **미검증 수치 표시**: 일정·스펙 등 담당 확인 필요한 값은 `[확인 필요]` 마커 부착.
4. **개인정보 마스킹**: `client_pm` 등 연락처는 입력·저장 단계에서 마스킹. 실명 대신 직무·등급 권장.
5. **누락 처리**: A 그룹 미입력 시 되묻기, B 그룹 미입력 시 기본값 적용 후 안내.
6. **고객 측 연락처 신설**: 7장 비상연락망에 고객 측 칸이 없으므로, 클로잇 측 형식
   (`{소속} {직책} [{이름} {직급} / {연락처} / {이메일}]`)을 따른 행을 **새로 추가**해 `client_pm`을 주입.
7. **자유 문단 교체**: 1장 검증 대상 솔루션(`solution_config`)은 표가 아닌 문단이므로
   셀 주입이 아니라 **문단 텍스트 전체를 재작성**해 교체한다(스타일 유지).

## 3. MVP 범위 (개발 직행)
- 대상: **A 7개 입력 → 1·2·3·5·6장 가변 항목 자동 작성 → 표준 .docx 출력**
- 후순위: B 그룹 되묻기 UI, 수정·재생성, 빠진 값 체크
- 라이브러리: python-docx 등 오픈소스(.docx 표·셀 주입) 전제
